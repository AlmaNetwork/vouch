# RFCs

Design discussions for the project, versioned and reviewed via pull request. Discussion
happens inline in each RFC's PR. An RFC is a proposal: nothing it describes is implemented
until agreed and landed separately.

Three buckets keep the design legible:

- **Procedure (source-of-truth)** — what counts as a binding decision. *Set* by founders.
  → RFC 0001.
- **Participation channels / affordances** — e.g. Voice (dissent paths for the governed),
  exit cost. *Set* by founders. → separate design (not yet drafted).
- **Observation** — the regime that emerges, **measured only** (never configured).
  → RFC 0002.
- **Interoperability & value (cross-region)** — how regions hold and transfer value, connect to
  one another, negotiate signatures, and authorize acts across borders. → RFC 0003–0006.
